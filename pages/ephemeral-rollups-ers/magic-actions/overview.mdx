## **Что такое Magic Actions?**

Magic Actions позволяют прикреплять одну или несколько инструкций вызова, которые выполняются автоматически на базовом слое Solana сразу после коммита в Ephemeral Rollup (ER). Используйте зафиксированное состояние в качестве входных данных для организации многошаговых рабочих процессов в одной транзакции.

- **Синхронизация состояния:** распространяйте изменения ER на аккаунты базового слоя
- **Композиция между программами:** вызывайте другие программы после коммита
- **Автоматизированные рабочие процессы:** цепочка сложных операций из одного коммита
- **Событийно-ориентированные действия:** ветвление логики на основе зафиксированного состояния

**Как это работает:**

1. Делегируйте аккаунты в Ephemeral Rollup
2. Выполняйте транзакции с низкой задержкой в Ephemeral Rollup
3. Коммитьте на базовый слой с прикреплёнными инструкциями действий, оставаясь делегированным
4. Обработчики выполняются автоматически, используя только что зафиксированное состояние
