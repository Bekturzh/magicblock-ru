## Поиск задач для работы

Если вы хотите начать, ознакомьтесь с открытыми задачами (issues). Для простых изменений документации или исправления опечаток можно сразу открыть pull request.

Если вы планируете более серьёзные изменения или хотите добавить новые функции по собственной инициативе, создайте issue и взаимодействуйте с сопровождающими проекта через Discord.

## Выбор задачи

Если вы хотите внести вклад, возьмите задачу на себя, оставив комментарий, сделав форк и открыв pull request, даже пустой. Это позволяет сопровождающим отслеживать, кто над какой задачей работает, чтобы избежать дублирования работы.

## Руководство по работе с задачами

**Перед кодированием:**

- Выберите имя ветки, которое описывает задачу, над которой вы работаете.

**Во время кодирования:**

- Как можно раньше отправьте draft PR (черновой pull request).
- Изменяйте только тот код, который напрямую относится к вашему PR. Иногда вы можете найти код, который требует рефакторинга, но если он не относится к вашему PR, не трогайте его. Вместо этого создайте отдельное issue. Это позволяет рецензенту сосредоточиться на одной проблеме за раз.
